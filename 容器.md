 # 1. <span style="color:red;">向量 vector
   ```#include <vector>  ``` 

    数组一样类型的存储结构，但长度可变

  * ## 1.1.基本构造
      ```c++
    vector<int> arr1;               //一个int类型的空数组
    vector<int> arr2(100)           //一个int类型的100个元素的数组
    vector<int> arr3(100,1);        //一个int类型的100个元素的数组，初值为1   
    vector<vector<int>> mat(100)    //初始100行的二维数组，未指定列数，不能直接用，会超出界限
    vector<vector<int>> mat(100,vector<10，1>)   //初始100行10列的二维数组，初值为1
    vector<vector<vector<int>>> dp(5,vector<vector<int>> (4,vector<int> (3,1)))    //一个等价于dp[5][4][3]的三维数组，初值为1
    ```
    对于未指定的行数列数的vector不能进行访问操作
  
  * ## 1.2.部分基本用法

    *```.push_back(元素)``` , ```.pop_back()```进行增删操作*

    *```.size()```获取长度*

    *```.clear()```清空vector，```.empty()```判断vector是否为空*

    *```.resize()```直接更改长度，若为扩大，则新元素均为默认值（原有的不变）*

    *```.insert(迭代器，值)```在迭代器指向的位置前插入元素*

    *```.erase(迭代器)/erase(起始迭代器,结束迭代器)```删除迭代器指向的元素*

      * *迭代器：类似下标，包括 ```.begin()``` ```.end()```和遍历出的 ```auto it```* 

    ```c++
    vector<int> arr;
    arr.push_back(1);   //arr[0]为1
    arr.push_back(2);   //arr[1]为2
    arr.push_back(3);   //arr[2]为3
    arr.pop_back();     //arr[2]在此时被删除
    arr.size();         //返回arr的长度，此时为2
    arr.clear();        //清空arr
    arr.empty();        //若arr为空返回1(true)，不为空则返回0(false)，此时为1
    arr.resize(3,10);   //更改arr长度为3，且元素均为默认值10
    arr.erase(arr.begin()+1); //删除了"1"位(第二位)
    arr.insert(arr.end()-1);  //删除了最后一位
    ```
  * ## 1.3.注意事项
 
      ### 1.3.1）提前指定长度
      尽量在构造函数时指定长度，而不是一个一个 ```.push_back()``` ，这样会浪费很多时间开销
      ### 1.3.2）注意 size_t 溢出
      使用```.size()```获取长度的返回值为 size_t 类型，在OJ平台中一般的32位系统，即size_t的取值范围是$[0,2^{32})$

      * *注意区别 size_t 和 int 类型的区别，size_t为$[0,2^{32})$，int为$(-2^{31},2^{31})$*

 # 2. <span style="color:red;">栈 stack
   ```#include <stack>```

   通过二次封装双端队列容器，实现 **先进后出** 的栈数据结构

  * ## 2.1构造
    ```c++
    stack<int> stk; //一个 int 类型的栈 stk
    ```
  * ## 2.2基本用法
    |作用|用法|示例|
    |:-:|:-:|:-|
    |进栈|```.push(元素)```|``` stk.push(1) //加入1到栈顶```|
    |出栈|```.pop()```|```stk.pop()    //将栈顶的元素删除```|
    |取栈顶|```.top()```|```stk.top()  //查看栈顶元素```|
    |查长度|```.size()```|```cout<<stk.size();  //返回值为这个栈的长度```|
    |判空|```.empty()```|```if(stk.empty())  //若为空返回1，否则为0```|
  * ## 2.3注意事项
    ### 2.3.1） vector 可模拟栈的用法
      使用 ```vector.back``` 相当于取栈顶， ```.push_back()``` 相当于进栈， ```.pop_back()``` 相当于出栈， stack 默认由 vector 实现
    ### 2.3.2） **不可访问内部元素**
      stack只允许访问```stk.top()```栈顶元素，遵循先进后出，无法访问中间和前面的元素
    ### 2.3.3） stack 自身没有```.clear()```用法（后面的quene也一样）
      要一次性清除所有元素，要用```stack<int>().swap(stk)```

 # 3. <span style="color:red;">队列 quene
 ```#include <quene>```

 通过二次封装双端对列容器，实现 **先进先出** 的队列数据结构
  * ## 3.1构造
    ```c++
    quene<int> que; //一个 int 类型的队列 que
    ```
  * ## 3.2基本用法
    |作用|用法|示例|
    |:-:|:-:|:-|
    |进队|```.push(元素)```|``` que.push(1) //加入1到队尾```|
    |出队|```.pop()```|```que.pop()    //将队首的元素删除```|
    |取队首|```.front()```|```que.front()  //查看队首元素```|
    |取队尾|```.back()```|```que.back();  //查看队尾元素```|
    |~~判空，取长度~~|~~同上~~|~~同上~~|
  * ## 3.3注意事项
    * **quene注意事项与stack基本一致，包括中间不可访问，需要```quene<int>().swap(que)```来实现清除元素等**

 # 4. <span style="color:red;">优先队列 priority quene（堆）
 ```include <quene>```

 提供 **常数** 时间的最大元素查找， **对数** 时间的插入与提取，底层原理是 **二叉堆**
  * ## 4.1构造
    ```c++
    priority_quene<类型，容器，比较器>pque;
    ```
    * 类型：要储存的数据类型
    * 容器：储存数据的底层容器，默认为```vector<类型>```，竞赛一般保持默认
    * 比较器：比较大小使用的比较器，默认为```less<类型>```，可自定义
    ```c++
    priority<int> pque1;                          //储存int的大顶堆（堆顶为最大数）
    priority<int,vector<int>,greater<int>>pque2   //储存int的小顶堆（堆顶为最小数）
    ```
  * ## 4.2基本用法
    |作用|用法|示例|
    |:-:|:-:|:-|
    |进堆|```.push(元素)```|```pque.push(1) //加入1到队尾```|
    |出堆|```.pop()```|```pque.pop()    //将队首的元素删除```|
    |取堆顶|```.top()```|```pque.top()  //查看队首元素```|
    |~~判空，取长度~~|~~同上~~|~~同上~~|
    ---
    *进出堆复杂度$O(\log{n})$，取堆顶复杂度$O(n)$*

    * 每次向堆中插入大小不定的元素，或去除最大最小的元素，元素数量n，插入操作数k
      * 每次插入后进行快排：$k*n*\log{n}$
      * 使用优先队列维护：$k*\log{n}$

  * ## 4.3注意事项
    ### 4.3.1）仅堆顶可读
      仅可访问堆顶，其他元素都不可读到

      ~~```cout << pque[1] << endl ;```~~```//错误示范```

    ### 4.3.2）所有元素不可修改（包括堆顶）
      与stack可直接修改栈顶```.top()```，quene可直接修改队首```.front()```尾```.back()```不同

      priority quene只能通过```.push()```和```.pop```间接修改堆顶元素 
      
      *（新加入的元素加完后不一定是堆顶）*
